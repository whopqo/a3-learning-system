# EduSynth · AI 个性化学习助手

> **第十五届中国软件杯大赛 · A组赛题**（A3-基于大模型的个性化资源生成与学习多智能体系统） | 出题企业：科大讯飞股份有限公司

基于大模型的多智能体学习系统：7 个专业 Agent 通过有向图协同，画像→路径→资源→答疑→评估→复盘形成教学闭环。学生用自然语言聊天，系统自动了解学习情况、定制学习路径、生成 8 种学习资料、答疑出题并跟踪掌握度。

---

## 快速开始

### 环境要求

- Windows / macOS / Linux
- Python 3.10 及以上（推荐用 conda 建独立环境）
- 一个大模型 API 密钥（推荐 [DeepSeek](https://platform.deepseek.com)，注册即送额度；也支持讯飞星火/通义千问/智谱/Kimi/Claude 等）
- 首次运行需要联网（自动下载约 100MB 的中文向量模型）

### 方式一：命令行（三条命令）

```bash
git clone https://github.com/whopqo/a3-learning-system.git
cd a3-learning-system

pip install -r requirements.txt        # 1. 安装依赖（几分钟）
python knowledge_base/build_kb.py      # 2. 构建知识库（首次必跑，含模型下载）
python run.py                          # 3. 启动，浏览器自动打开 http://localhost:8000
```

### 方式二：IDE 右键运行（不想敲命令）

用 VS Code / PyCharm 打开项目文件夹：

1. 右键运行 `install.py` —— 自动装依赖 + 下载模型 + 构建知识库（等 3-5 分钟）
2. 右键运行 `run.py` —— 浏览器自动打开

### 配置 API 密钥（两种方式任选）

**方式 A：网页配置（推荐，不用碰文件）**

启动后打开网页 → **设置 → 模型配置** → 点一个服务商卡片（如 DeepSeek）→ 贴上密钥 → 点「测试连接」→ 点「保存并设为当前」。立即生效，不用重启。

**方式 B：配置文件**

把 `.env.example` 复制一份改名为 `.env`，用记事本打开填入：

```
DEEPSEEK_API_KEY=sk-你的密钥
```

> 密钥只保存在你自己的电脑上（`.env` 和 `db/` 都不会被提交到 Git）。

## 功能一览

| 功能 | 说明 |
|---|---|
| 对话式画像 | 聊天自动构建 8 维学习画像；开场有「零基础/有基础/进阶」快速开始按钮，一轮建好；支持手动修正，每个维度可查看 AI 判断依据 |
| 多智能体资源生成 | 8 种资源：讲义、思维导图（可折叠交互）、练习题、阅读材料、拓展阅读、代码案例、PPT（可导出 .pptx）、视频脚本（可渲染 MP4）；讲义自动标注知识库出处 |
| 学习路径规划 | 基于章节依赖图 + 画像定制的阶段式计划，零基础自动加预备阶段 |
| 智能答疑 | 4 级辅导策略（直接解答/提示引导/类比讲解/苏格拉底追问）+ 自动生成概念流程图 |
| 学习评估 | 秒判对错 + AI 深度错因分析；错题自动归类（概念混淆/计算失误/审题不清/知识空白），出题时针对性加强 |
| 掌握度地图 | 每个知识点 0~100% 分数持续演化，带遗忘曲线衰减，快忘的知识点提醒复习 |
| 错题本 + 每日一练 | 错题自动归档、一键生成变式复习卷；守护进程每天按薄弱点自动出题 |
| 难度自适应 | 按最近答题正确率自动升降出题难度 |
| 模型配置中心 | 10 家服务商热切换（DeepSeek/星火/千问/智谱/Kimi/Claude/硅基流动/OpenRouter/Ollama/自定义），OpenAI 与 Anthropic 双协议，聊天框可随时换模型 |
| 教学模式 | 用 SKILL.md 文件自定义 AI 教学风格（内置零基础引导/考研冲刺/苏格拉底引导），聊天时勾选或说话带关键词自动触发 |
| 章节依赖图 | 17 章知识结构可视化，已掌握章节标绿，支持放大查看 |
| 知识库管理 | 内置西瓜书 17 章教材；支持上传自己的 txt/md 资料入库，AI 之后基于你的资料回答 |
| 记录持久化 | 聊天记录、画像、答题记录关掉页面再开都在；右上角「重置」一键从头开始 |
| 系统运维 | 环境一键体检（绿勾红叉）、日志按天落盘可筛选、守护进程定时巡检 |

## 常见问题

| 问题 | 解决 |
|---|---|
| 启动时提示"未设置 API Key" | 正常，去网页 设置→模型配置 填密钥即可，不影响启动 |
| 第一次运行很慢 | 首次要下载约 100MB 向量模型，之后有缓存秒开 |
| 提示"知识库未构建" | 运行 `python knowledge_base/build_kb.py` |
| 聊天没反应/报错 | 设置→模型配置 点「测试连接」检查密钥；设置→系统日志 看 ERROR |
| 生成资料要等一会儿 | 正常，8 种资源并行调用大模型，约 15-30 秒 |
| 端口 8000 被占用 | 关掉其他占用程序，或任务管理器结束多余的 python 进程 |

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | HTML5 + 原生 JS（ECharts 数据图表 / Markmap 思维导图 / Mermaid 流程图） |
| 后端 | FastAPI + Uvicorn（REST + SSE 流式输出） |
| 多智能体 | 自研 AgentGraph 有向图引擎（节点 = Agent + 质量门，不合格自动回退重试） |
| LLM 接入 | OpenAI 兼容协议 + 自研 Anthropic 协议翻译器，10 家服务商热切换 |
| RAG | ChromaDB + BAAI/bge-small-zh-v1.5 本地向量化，全部生成内容强制知识库溯源防幻觉 |
| 知识图谱 | NetworkX 章节依赖 DAG |
| 导出 | python-pptx 生成可编辑 PPT；Manim 渲染讲解视频（可选） |
