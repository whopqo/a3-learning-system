## 一、需求分析

### 1.1 业务背景

在高等教育场景中，学生面临三大核心痛点：

1. **资源匹配困难**：海量课程资料、学术文献纷繁杂序，学生难以快速筛选出契合自身进度的学习内容
2. **教学个性化不足**：标准化课堂无法兼顾每位学生的知识基础、认知风格和学习节奏
3. **学习缺乏引导**：传统智能辅导系统缺乏多模态生成与多智能体协同能力，无法实现"因材施教"

### 1.2 用户画像

本系统的核心用户为**高校本科/研究生**，以计算机、人工智能、电子信息等专业为主。典型用户特征：

- 知识基础参差不齐（数学、编程水平各异）
- 学习风格多样（视觉型、动手型、听觉型、文字型）
- 学习目标分层（通过考试、掌握技能、独立做项目）
- 可用时间碎片化（45-60分钟/次，3-5次/周）

### 1.3 功能需求矩阵

| 编号 | 功能 | 优先级 | 创新点 |
|------|------|:---:|--------|
| F1 | 对话式学习画像构建（6+维度，随学随新） | P0 | 弃用表单，自然语言对话抽取特征 |
| F2 | 多智能体协同资源生成（7种类型） | P0 | 7个专业Agent分工协作 |
| F3 | 个性化学习路径规划 | P0 | 知识图谱拓扑排序+画像权重 |
| F4 | 智能辅导答疑（多模态解答） | P1 | 4级辅导策略+图解生成 |
| F5 | 学习效果评估（闭环调整） | P1 | 评估→画像→路径完整闭环 |

### 1.4 非功能需求

| 编号 | 要求 | 实现方式 |
|------|------|---------|
| NF1 | 流式输出 + Markdown渲染 | SSE模拟 + Streamlit原生渲染 |
| NF2 | 生成进度追踪 | 每步进度回调 + 进度条 |
| NF3 | 防幻觉机制 | RAG溯源 + 事实校验 + 敏感过滤 |
| NF4 | 合理响应时间 | DeepSeek API（秒级）+ 进度可见性 |

## 二、系统架构

### 2.1 整体架构

```
┌──────────────────────────────────────────────────────┐
│                   前端展示层 (Streamlit)               │
│  对话面板 | 资源卡片 | 学习路径 | 画像仪表盘          │
├──────────────────────────────────────────────────────┤
│                多智能体协同层 (7 Agent)                │
│  ┌─────────────────────────────────────────────┐    │
│  │  Supervisor (协调者)                         │    │
│  │  ├── ProfileBuilder (画像构建)               │    │
│  │  ├── ResourceGenerator (资源生成，7种子Agent) │    │
│  │  ├── PathPlanner (路径规划)                  │    │
│  │  ├── SmartTutor (智能辅导)                   │    │
│  │  ├── Evaluator (学习评估)                    │    │
│  │  └── MemoryManager (记忆管理，L1/L2/L3)      │    │
│  └─────────────────────────────────────────────┘    │
├──────────────────────────────────────────────────────┤
│              共享服务层                               │
│  RAG引擎 | 知识图谱 | 内容安全 | 解析器              │
├──────────────────────────────────────────────────────┤
│              模型层                                   │
│  DeepSeek API (主) / 科大讯飞Spark (备用)            │
│  BGE Embedding (本地向量化)                          │
├──────────────────────────────────────────────────────┤
│              数据层                                   │
│  ChromaDB (向量) | JSON文件 (记忆) | NetworkX (图谱) │
└──────────────────────────────────────────────────────┘
```

### 2.2 多智能体协作机制

系统采用**Supervisor协调者模式**，参考LangGraph和CrewAI的设计思想：

1. **意图识别**：关键词快速匹配（毫秒级），省去每次调用LLM的开销
2. **路由分发**：根据意图将任务分配给最合适的Agent
3. **并行执行**：资源生成阶段，7种资源子Agent并行调用
4. **闭环反馈**：评估结果→自动更新画像→自动调整学习路径

### 2.3 核心数据流

```
用户输入 → 安全检查 → 意图识别 → 路由到Agent
                                    ↓
资源生成Agent: 检索RAG → 并行生成7种资源 → 审核 → 存储
路径Agent: 查询知识图谱 → 个性化评分 → 生成路径
辅导Agent: 分析问题 → RAG检索 → 解答 + 图解
评估Agent: 逐题批改 → 掌握度计算 → 更新画像 → 调整路径
```

## 三、核心技术设计

### 3.1 对话式学习画像

**设计理念**：弃用传统表单采集，通过自然语言对话自动提取6+维度特征。

**画像维度**：
1. 知识基础（math/programming/ML前置评分）
2. 认知风格（视觉/听觉/动手/文字型）
3. 易错点偏好（错误模式+类型分类）
4. 学习目标（短期/中期/长期）
5. 学习投入度（单次时长+每周频率+完成率）
6. 资源偏好（格式偏好+难度级别）
7. 动态学习历史（已掌握+薄弱+近期进步）

**实现方式**：每次对话后自动调用LLM抽取特征→更新画像→判断是否需要更多信息→生成引导问题。

### 3.2 RAG驱动的资源生成

**五步RAG管道**（参考Verba和RAG_Techniques）：
1. **文档摄取**：课程教材txt文件
2. **语义分段**：500字符/块，50字符重叠
3. **向量化**：BGE-small-zh-v1.5 (512维)
4. **混合检索**：向量语义 + 元数据过滤，Top-K=5
5. **生成**：基于检索结果+学生画像的个性化内容生成

**7种资源类型**：
| 类型 | 格式 | 生成方式 |
|------|------|---------|
| 课程讲义 | Markdown | LLM + RAG + 结构化大纲 |
| 思维导图 | Mermaid代码 | LLM + 知识图谱 |
| 练习题 | JSON题库 | LLM + Bloom分类法 |
| 拓展阅读 | 推荐列表 | LLM + RAG检索 |
| 代码案例 | Python代码 | LLM + sklearn示例 |
| PPT大纲 | Markdown | LLM + 结构化格式 |
| 视频脚本 | 分镜+旁白 | LLM + 场景描述 |

### 3.3 知识图谱驱动的路径规划

**知识图谱**：基于NetworkX构建的机器学习课程有向无环图（DAG），包含20个知识点节点和30+条前置依赖边。

**路径规划算法**（贪心+启发式评分）：
```
Score(节点) = 0.30×难度匹配 + 0.25×资源可用性 + 0.25×目标相关度 + 0.20×前置满足度
```

**动态调整**：评估结果中发现的新薄弱点 → 自动插入复习节点 → 重新排序路径。

### 3.4 掌握度算法（参考DeepTutor）

```python
权重 = (0.5, 0.7, 0.85, 0.95, 1.0)  # 越新的答题权重越高
置信度封顶: 1次最多0.5, 2次最多0.8   # 防止蒙对算掌握
掌握度 = 最近N次答题的加权平均（受置信度封顶限制）
```

### 3.5 防幻觉三层机制

1. **输入层**：敏感词过滤 + 注入攻击检测 + 范围校验
2. **生成层**：RAG强制溯源 + 低温度(0.2-0.4) + 来源标注
3. **输出层**：事实一致性校验 + 关键词重叠检测 + 幻觉风险评估

## 四、创新实践

### 4.1 技术创新

1. **轻量级多智能体框架**：不依赖LangChain/CrewAI等重量框架，自建7个Agent的Supervisor协同模式，代码量精简至3600+行

2. **知识图谱+LLM双驱动**：LLM负责理解和生成，知识图谱负责逻辑和结构，互补增强

3. **三层记忆系统**：L1原始交互→L2知识状态→L3学习策略，层层抽象，支持追溯

4. **评估→画像→路径闭环**：评估结果自动反哺画像和路径，实现真正的"随学随新"

### 4.2 用户体验创新

5. **全流式交互**：逐token输出+每步生成进度追踪，消除白屏等待焦虑

6. **卡片化资源展示**：7种资源类型差异化渲染（代码高亮/Mermaid渲染/交互答题）

7. **一键操作**：快捷按钮减少输入负担，"学习路径"、"做练习"等常用操作一键直达

## 六、部署与运行

### 6.1 环境要求
- Python 3.11+
- Conda（推荐）
- DeepSeek API Key（或其他OpenAI兼容API）

### 6.2 启动步骤
```bash
conda activate test
cd a3-learning-system
pip install -r requirements.txt        # 安装依赖
python knowledge_base/build_kb.py      # 构建知识库（首次）
python run.py                          # 启动系统
```

### 6.3 配置切换
修改 `config.py` 中的三行即可切换大模型：
```python
LLM_API_KEY = "your-key"
LLM_BASE_URL = "https://api.xxx.com/v1"
LLM_MODEL = "model-name"
```
