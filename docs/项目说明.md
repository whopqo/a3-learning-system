# A3 AI 学习助手 — 项目说明

> **第十五届中国软件杯大赛 · A组赛题**  
> **A3-基于大模型的个性化资源生成与学习多智能体系统开发**  
> 出题企业：科大讯飞股份有限公司

---

## 一、项目概述

一个基于大模型的个性化学习多智能体系统。学生通过自然语言对话让系统了解自己的学习情况，系统自动构建学生画像，生成个性化学习路径和 8 种学习资源，并提供智能答疑、练习评估等闭环服务。

核心思路：7 个专业 Agent 通过有向图协同工作，每个 Agent 负责一个专业领域（画像分析、路径规划、资源生成、辅导答疑、学习评估、记忆管理、全局协调），Agent 之间通过共享上下文协作，评估结果自动反馈到画像和路径形成闭环。

## 二、技术架构

### 整体四层架构

```
┌────────────────────────────────────────────────┐
│              前端展示层                          │
│  HTML5 + 原生 JavaScript (SPA 单页应用)         │
│  Mermaid 图表渲染 · Canvas 雷达图/折线图        │
│  SSE 流式接收 · Markdown 渲染 · 6 页面标签      │
├────────────────────────────────────────────────┤
│              后端服务层                          │
│  FastAPI + Uvicorn (REST API + SSE 流式)       │
│  handler.py: 消息收发 + 会话管理                │
│  server.py: 12 个 API 端点                     │
├────────────────────────────────────────────────┤
│           多智能体协同层 (AgentGraph)            │
│  ┌──────────────────────────────────────┐     │
│  │  EduSupervisor (中央协调者)            │     │
│  │  ├── ProfileBuilder  (画像构建)       │     │
│  │  ├── PathPlanner     (路径规划)       │     │
│  │  ├── ResourceGenerator (资源生成)     │     │
│  │  ├── SmartTutor      (智能辅导)       │     │
│  │  ├── Evaluator       (学习评估)       │     │
│  │  └── MemoryManager   (记忆管理)       │     │
│  └──────────────────────────────────────┘     │
│  有向图引擎：节点=Agent+质量门，边=通过/重试    │
├────────────────────────────────────────────────┤
│              基础设施层                          │
│  LLM: DeepSeek API (OpenAI兼容，多Provider切换) │
│  Embedding: BAAI/bge-small-zh-v1.5 (本地)     │
│  向量库: ChromaDB (持久化)                     │
│  知识图谱: NetworkX (有向图 DAG)                │
│  PPT 导出: python-pptx                         │
│  视频渲染: Manim (可选)                         │
└────────────────────────────────────────────────┘
```

### 多智能体有向图协同

```
profile ──(够6维?)→ path ──(有阶段?)→ resource ──(KB匹配?)→ done
  ↑ 不够                ↑ 无阶段           ↑ 不匹配
  └── 继续追问          └── 重新规划        └── 换知识点重试

evaluate ──(掌握≥50%?)→ done
  ↓ 没掌握
resource ←── 出针对性练习题
profile ←── 更新画像薄弱点
path ←── 调整学习路径
```

每个节点后有质量门检查，不通过回到上游重试。评估→画像→路径形成自反馈闭环。

## 四、赛题功能实现对照

### F1：对话式学习画像构建（≥6 维度）

**实现方式**：LLM 驱动的自然对话，不使用任何表单。

每轮对话：
1. `ProfileAgent.generate_next_question()` 根据已收集的信息，用 LLM 生成一句自然的引导问题
2. 设置 `_current_dim` 标记本轮在问哪个维度
3. 学生回答后，`extract_profile_update()` 只提取当前维度（LLM 打分/分类）
4. 标记已问、计数更新次数

**采集的 8 个维度**：

| 维度 | 字段 | 说明 |
|--|--|--|
| ML 基础 | `ml_prerequisites` | 对机器学习的了解程度（0.15-0.9 评分） |
| 编程水平 | `programming` | Python 代码能力（0.15-0.9 评分） |
| 数学基础 | `math` | 线代/概率论水平（0.15-0.9 评分） |
| 学习目标 | `short_term_goal` | 考研/就业/考试/兴趣 |
| 认知风格 | `cognitive_style` | 视觉型/文字型/动手型/听觉型 |
| 时间投入 | `time_per_week` | 每周学习小时数 |
| 语言偏好 | `language_style` | 严谨学术/生动比喻，影响讲义表达方式 |
| 薄弱点 | `struggling_topics` | 觉得难的知识点 |

**防重复机制**：四重防护确保不会提前结束——维度追踪、更新计数、轮数下限、维度数下限。

---

### F2：多智能体协同资源生成（≥5 种类型）

**架构**：有向图引擎 `agents/graph.py`（369 行）。`AgentGraph` 定义 5 个节点（profile/path/resource/tutor/evaluate），每个节点 = Agent + 质量门。`walk()` 方法沿图遍历，yield SSE 事件实现流式输出。

**生成的 8 种资源**：

| 资源 | 生成方式 | 个性化依据 |
|--|--|--|
| 讲解文档 | LLM + RAG + 画像`_emphasis`策略 | 基础弱→概念类比，基础好→数学推导 |
| 思维导图 | LLM→JSON→Mermaid | 基于 RAG 知识库内容 |
| 练习题 | LLM + 画像`_emphasis` | 零基础5题/一般7题/进阶8题，薄弱点多加概念题 |
| 阅读材料 | 知识库原文提取 | 只取当前章节的教材推荐 |
| 拓展阅读 | 知识库"休息一会儿"小故事 | 从17章中抽取 |
| 代码案例 | LLM + 画像编程水平 | 新手→30行带详细注释，熟手→60行 |
| PPT 大纲 | LLM + RAG | 15-20页结构化，可导出.pptx |
| 视频脚本 | LLM + Manim模板 | 简化模板保证可渲染 |

**防幻觉机制**：所有资源生成 prompt 注入 `KB_ONLY_RULE`，强制基于知识库原文，不得编造。生成前先检查知识点是否在知识库中。

**知识库**：17 章完整西瓜书课程内容，通过 ChromaDB 向量化存储（180 个语义片段），BGE Embedding 编码。

---

### F3：个性化学习路径规划和资源推送

**实现方式**：`PathAgent` 根据画像用 LLM 生成阶段式学习计划。

**路径生成流程**：
1. 将画像转为人读的摘要文本（基础评分/目标/风格/薄弱点）
2. 从知识库检索课程大纲
3. 从知识图谱获取全部章节列表
4. LLM 根据画像生成阶段式路径 JSON：阶段名称、时长、目标、涵盖章节、动手任务、推荐资源
5. 零基础学生自动加入预备知识阶段，进阶学生压缩基础阶段

**不是固定的 16 章列表**，而是根据画像定制的阶段计划（通常 4-6 个阶段）。

**资源推送**：生成资源时自动匹配所属阶段和路径步骤，输出个性化描述（基础评估+目标方向+学习风格+薄弱提醒）。

---

### F4：智能辅导答疑（加分项）

**实现方式**：`TutorAgent` 4 级辅导策略 + RAG 检索。

| 等级 | 策略 | 适用场景 |
|--|--|--|
| L1 | 直接解答 | 学生明确要求 |
| L2 | 提示引导 | 先给线索再给答案 |
| L3 | 类比讲解 | 生活化比喻+技术解答 |
| L4 | 苏格拉底追问 | 递进问题引导自悟 |

**多模态解答**：文字 Markdown + Mermaid 图解 + Manim 动画解释。知识库外问题直接拒绝。

---

### F5：学习效果评估（加分项）

**实现方式**：`EvaluateAgent` LLM 深度分析 + 关键词快速判对错。

**评估流程**：
1. 关键词毫秒级判对错（立即显示得分）
2. `EvaluateAgent.process()` LLM 异步深度分析错因
3. 掌握度计算（参考 DeepTutor 加权算法，recency weights + 置信度封顶）
4. 自动更新画像薄弱点 + 已掌握列表
5. 触发路径调整（评估→画像→路径闭环）

**两种评估入口**：
- 弹窗 Quiz（资源库→练习题→提交评估）
- 聊天评估（对话中直接发送答案）

## 六、每个文件干什么（小白也能看懂）

### 入口文件（放在最外层，一打开就能看到）

| 文件 | 用大白话说 |
|------|-----------|
| `run.py` | **启动器**。在 VS Code 里右键点"运行 Python 文件"，系统就启动了，浏览器会自动打开。不用去终端敲命令。 |
| `install.py` | **安装器**。第一次用的时候右键运行它，它会自动装好所有依赖、下载 AI 模型、构建知识库。等 3-5 分钟就行。 |
| `server.py` | **后端服务器**。接收浏览器发来的请求（比如你发的聊天消息），调用 AI 处理，把结果流式返回给浏览器。整个系统的"门面"。 |
| `handler.py` | **总调度**。server.py 收到消息后交给它，它判断你要干什么（聊天？出题？答疑？），然后叫对应的 AI 角色来处理。相当于公司的前台。 |
| `config.py` | **配置中心**。所有设置都在这里：用哪个大模型（DeepSeek/讯飞星火/通义千问）、API 密钥、知识库路径、安全关键词。改配置只改这一个文件。 |
| `requirements.txt` | **购物清单**。列出了这个项目需要安装的所有 Python 库，`pip install -r requirements.txt` 就是照着这个清单买东西。 |
| `.env.example` | **密钥模板**。复制它改名为 `.env`，在里面填你的 DeepSeek API 密钥。`.env` 不会被提交到 GitHub，密钥安全。 |
| `README.md` | **项目首页**。GitHub 上别人打开你的仓库第一眼看到的介绍。 |
| `项目说明.md` | **就你现在看的这个**。给评审老师和小白看的完整说明书。 |

### 前端界面（用户看到的网页）

| 文件 | 用大白话说 |
|------|-----------|
| `static/index.html` | **整个网页**。你用浏览器打开的 http://localhost:8000 就是这个文件。包含了 7 个标签页（对话/画像/路径/资源库/知识库/记录/设置）的全部代码。聊天泡泡、Mermaid 图表、Canvas 雷达图、SSE 流式接收都在这里面。聊天输入框旁边有模型下拉框，可以随时切换用哪个大模型回答。 |

### 智能体模块（agents/ 目录——7 个 AI 角色 + 1 个调度图）

| 文件 | 用大白话说 |
|------|-----------|
| `agents/graph.py` | **有向图引擎**。7 个 AI 角色的协作流程图。定义了谁先干活、干完后交给谁、质量不合格退回重来。是整个"多智能体协同"的核心。 |
| `agents/base_agent.py` | **AI 角色的爹**。所有 AI 角色的公共部分：调用大模型、自动重试（失败了重来 3 次）、从 AI 返回的文本里提取 JSON。 |
| `agents/supervisor.py` | **协调者**。总管角色，负责判断用户意图、做 LLM 分类、闲聊回应。其他 AI 角色都通过它来调用大模型。 |
| `agents/profile_agent.py` | **学情分析师**。通过对话了解你的水平，构建 8 维学习画像。LLM 自然发问，每轮记录一个维度，8 维全问过才放行。 |
| `agents/resource_agent.py` | **课件生成师**。根据画像和知识库，生成 8 种学习资料：讲义、思维导图、练习题、阅读材料、拓展阅读、代码案例、PPT 大纲、视频脚本。 |
| `agents/path_agent.py` | **路径规划师**。根据你的画像，用 LLM 生成阶段式个性化学习路径。不是固定的 16 章列表——零基础有预备阶段，进阶跳入门章。 |
| `agents/tutor_agent.py` | **辅导老师**。你问概念它来答。4 级辅导策略（直接解答/提示引导/类比讲解/苏格拉底追问），还带 Mermaid 图解。 |
| `agents/evaluate_agent.py` | **评估师**。批改你的练习题，计算掌握度，分析错因。用的算法参考了 DeepTutor 的加权公式，越新的答题权重越高。 |
| `agents/memory_agent.py` | **记忆管理**。三层记忆系统：L1 存原始聊天记录、L2 存知识点掌握情况、L3 存学习策略建议。 |

### 知识检索（rag/ 目录——让 AI 知道该讲什么）

| 文件 | 用大白话说 |
|------|-----------|
| `rag/engine.py` | **RAG 检索引擎**。把 17 章西瓜书教材切成 500 字的小段，用 AI 转成向量存起来。你问问题的时候，它找出最相关的几段给大模型做参考——这样 AI 就不会瞎编了。 |

### 工具模块（utils/ 目录——各种帮手）

| 文件 | 用大白话说 |
|------|-----------|
| `utils/content_safety.py` | **安全审查员**。检查你输入的内容有没有敏感词（暴力、色情等），有没有人试图攻击系统（让 AI 忽略指令）。三道防线。 |
| `utils/knowledge_graph.py` | **知识图谱**。用 NetworkX 画出 17 章教材之间的前置依赖关系（比如第 3 章"线性模型"必须在第 1 章"绪论"之后学）。还从 cross_edges.json 加载跨章节依赖。 |
| `utils/parser.py` | **JSON 提取器**。大模型返回的内容有时候带着 markdown 标记和废话，这个工具负责把纯 JSON 从里面扒出来。 |
| `utils/ppt_exporter.py` | **PPT 导出工具**。把 AI 生成的 PPT 大纲转成真正的 .pptx 文件。深蓝底色+金色装饰线+微软雅黑字体，可以直接用。 |
| `utils/llm_manager.py` | **模型管理中心**。内置 9 家大模型服务商的地址（DeepSeek/星火/千问/智谱/Kimi/Claude/硅基流动/OpenRouter/Ollama）+自定义接口。在设置页填密钥、测连接、切模型，**切换立即生效不用重启**。普通配置存 `db/models.json`，密钥单独存 `db/secrets.json`（不会被提交到 GitHub）。 |
| `utils/anthropic_compat.py` | **协议翻译器**。Claude 官方接口和 OpenAI 格式不一样（地址、密钥位置、返回结构都不同），这个翻译器把差异抹平，Claude 格式的接口也能无缝接入。测试连接时还能自动识别对方是哪种协议。 |
| `utils/skill_manager.py` | **教学模式管理**。一个模式 = `skills/模式名/SKILL.md` 文件（给 AI 的教学规则）。聊天时点「模式」按钮勾选，或者说话带触发关键词自动启用。内置 3 个：零基础引导、考研冲刺、苏格拉底引导。设置页→教学模式可以自己新建。 |
| `utils/mistake_book.py` | **错题本**。答错的题自动存进 `db/mistakes.json`（带知识点标签）。记录页能看错题列表，点「生成复习卷」会针对错题出新的变式题（不是原题）帮你巩固。 |

### 有向图引擎 (`agents/graph.py`)

多Agent协同的核心——每个节点 = Agent + 质量门，环 = 不通过时回到上游：

```python
class AgentGraph:
    MAX_LOOPS = 5

    def _build(self):
        self.nodes["profile"]  = GraphNode("profile", ..., on_pass="path", on_fail="profile")
        self.nodes["path"]     = GraphNode("path", ..., on_pass="resource", on_fail="path")
        self.nodes["resource"] = GraphNode("resource", ..., on_pass="done", on_fail="resource")
        self.nodes["tutor"]    = GraphNode("tutor", ..., on_pass="done")
        self.nodes["evaluate"] = GraphNode("evaluate", ..., on_pass="done", on_fail="resource")

    def walk(self, start: str, ctx: dict) -> Generator[dict, None, dict]:
        current = start
        while current and current != "done":
            # 执行节点 → 流式输出
            yield from self._run_node(node, ctx)
            # 质量门检查
            passed, _ = self._check_node(node, ctx)
            current = node.on_pass if passed else node.on_fail
```

## 八、第一次运行

### 环境要求
- Windows 10/11 或 macOS/Linux
- Python 3.10+
- DeepSeek API Key（或其他 OpenAI 兼容接口）

### 启动步骤

**方式一：一键安装（推荐）**
1. 安装 Python 3.10+（勾选 Add to PATH）
2. 复制 `.env.example` 为 `.env`，填入 `DEEPSEEK_API_KEY=你的Key`
3. 右键 `install.py` → 运行 Python 文件 → 等待 3-5 分钟
4. 右键 `run.py` → 运行 Python 文件 → 浏览器自动打开

**方式二：手动**
```bash
pip install -r requirements.txt          # 安装依赖
python knowledge_base/build_kb.py        # 构建知识库（首次必须）
python run.py                            # 启动
```

浏览器访问 http://localhost:8000

### 切换大模型

不用改代码了！打开网页 → **设置 → 模型配置**：
1. 点一个服务商卡片（比如 Kimi）
2. 贴上你的 API 密钥，点「测试连接」（成功会自动拉取可选模型列表）
3. 点「保存并设为当前」→ 立即生效，不用重启

聊天时也可以临时换模型：聊天输入框左边有个下拉框，选哪个模型，下一条消息就用哪个回答。

## 九、常见问题
